Lec 1

Computer Vision

  • light(simple point source or any other complex sytem) falls on a scene and gets scattered and reflected
  • camera captures this light by digitalizing the light intensity
    • sampling and quantization are involved
    • image is then represented as matrix of these values (pixels)
  • a computer vision derives some information from this and gives scence description

Image formation model

  • how a 3d object is projected onto a 2d image plane
  • we understand this to later reconstruct some information from a 2d image

Perspective projection with Pinhole

  • the image of a point PoP_oPo​ with position vector ro→=(xo,yo,zo)\overrightarrow{r_o} = (x_o, y_o, z_o)ro​​=(xo​,yo​,zo​) gets projected on the image plane at point PiP_iPi​ with position vector ri→=(xi,yi,f)\overrightarrow{r_i} = (x_i, y_i, f)ri​​=(xi​,yi​,f)
  • from the similarities of triangle we get:
    ri→f=ro→zo  ⟹  xif=xozo,yif=yozo\frac{\overrightarrow{r_i}}{f} = \frac{\overrightarrow{r_o}}{z_o} \implies \frac{x_i}{f} = \frac{x_o}{z_o}\hspace{2mm},\hspace{2mm} \frac{y_i}{f} = \frac{y_o}{z_o}fri​​​=zo​ro​​​⟹fxi​​=zo​xo​​,fyi​​=zo​yo​​
  • these equations are known as equations of perspective projection

Properties of perspective projection

  • straign line in the scene remains a straight line in the image
  • maginification mmm is give by m=fzom = \frac{f}{z_o}m=zo​f​
    • reason for point intersecting at infinity

Linear Camera Model

  • mapping the 3D points into 2D image plane
  • also known as forward imaging model
  • the image formed depends on:
    • Extrinsic Parameter - orientation of camera with respect to the object in the 3D world (world coordinate frame)
    • Intrinsic Parameter - how the camera maps the 3D coordinates of the object on an 2D image plan(internal parameters of the camera)
  • in the forward imaging model:
    • world coordinates of a point Xw=[xwywzw]TX_w = [x_w y_w z_w]^TXw​=[xw​yw​zw​]T(position vector with respect to the world coordinate system) gets transformed to camera coordinates Xc=[xcyczc]TX_c = [x_c y_c z_c]^TXc​=[xc​yc​zc​]T(position vector with respect to the camera coordinate system)
    • this camera coordinates get mapped on the image plane Xi=[xi,yi]TX_i = [x_i, y_i]^TXi​=[xi​,yi​]T using the perspective projection

Camera to image transformation

  • using the equations of perspective projection, we can find the relation between image coordinates and the camera coordinates
    xif=xczc,yif=yczc\frac{x_i}{f} = \frac{x_c}{z_c}\hspace{2mm},\hspace{2mm} \frac{y_i}{f} = \frac{y_c}{z_c}fxi​​=zc​xc​​,fyi​​=zc​yc​​

  • if we consider the density(pixels/mm) of pixels in the x^\hat{x}x^ and y^\hat{y}y^​ direction as mxm_xmx​ and mym_ymy​ and number of pizels be uuu and vvv, then we can relate them by:
    u=mxxi,v=myyiu = m_x x_i \hspace{2mm},\hspace{2mm} v = m_y y_iu=mx​xi​,v=my​yi​
  • using the equations between image and camera corrdiantes, we get
    u=mxfxczc,v=myfyczcu = m_x f \frac{x_c}{z_c} \hspace{2mm},\hspace{2mm} v = m_y f \frac{y_c}{z_c}u=mx​fzc​xc​​,v=my​fzc​yc​​
